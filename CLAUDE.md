## 🔴 مهم‌ترین قانون: استفاده از Code Graph

**برای هر گونه تحلیل کد، جستجو، فهم وابستگی‌ها و تغییر کد، باید از «code graph» (گراف کد) استفاده شود.**

- **در شروع کار (هر سشن):** ابتدا code graph را **سینک (sync)** کن تا با آخرین وضعیت کد هماهنگ باشد.
- **بعد از هر تغییری در کد:** دوباره code graph را **سینک کن** تا گراف به‌روز بماند و تحلیل‌های بعدی درست باشند.
- تحلیل ساختار، یافتن توابع/کلاس‌ها، ردیابی وابستگی‌ها و تأثیر تغییرات باید از طریق code graph انجام شود، نه صرفاً جستجوی متنی.

> خلاصه: **قبل از شروع → sync. بعد از تغییر کد → sync.** همیشه گراف کد را به‌روز نگه دار.

## قواعد کدنویسی (مهم)

- **کد و نام متغیر/تابع: انگلیسی.**
- **کامنت‌ها: فارسی** (تا مالک پروژه متوجه شود).
- **متن‌های رو‌به‌کاربر (پیام‌های ربات): فارسی.**
- از enumها و ثابت‌ها به‌جای رشته‌های سحرآمیز استفاده کن (`bot/enums.py`, `bot/constants.py`).
- منطق بازی در `services/` (مستقل از تلگرام، قابل‌تست)؛ هندلرها فقط لایه‌ی تلگرام.

## دستورهای اجرا

```bash
pip install -r requirements.txt        # نصب وابستگی‌ها
cp .env.example .env                    # پر کردن مقادیر واقعی
python -m scripts.seed                  # ساخت جدول‌ها + ۳۶ کشور (یک‌بار)
python -m bot                           # اجرای ربات
```

دستورهای کمکی:
```bash
python -m scripts.seed --refresh-military   # تازه‌سازی تجهیزات از countries.json روی DB موجود
python -m scripts.migrate                    # مهاجرت دستی ستون‌ها (init_db خودکار هم انجام می‌دهد)
```

## نکات دیتابیس و مهاجرت

- `init_db()` در `bot/database/base.py` هنگام راه‌اندازی، جدول‌های جدید را می‌سازد و
  **ستون‌های جدید را به‌صورت خودکار به جدول‌های موجود اضافه می‌کند** (لیست `_COLUMN_MIGRATIONS`).
- **هر بار که ستونی به یک مدل اضافه کردی، حتماً آن را به `_COLUMN_MIGRATIONS` هم اضافه کن**
  تا روی دیتابیس‌های قدیمی (production) هم خودکار اعمال شود.
- `countries.json` منبع حقیقت برای داده‌ی اولیه است؛ ریست فصل (`season_service`) از روی آن بازسازی می‌کند.

## تست (بدون نیاز به PostgreSQL/توکن واقعی)

برای تست منطقی، از SQLite درون‌حافظه‌ای/فایل با env آزمایشی استفاده کن:
```bash
BOT_TOKEN="123456:ABC-test" DATABASE_URL="sqlite+aiosqlite:///./test.db" \
GROQ_API_KEY="gsk_test" OWNER_IDS="111" ADMIN_IDS="111" PYTHONUTF8="1" python -c "..."
```
- همیشه `PYTHONUTF8=1` بگذار (کنسول ویندوز با ایموجی فارسی مشکل دارد).
- بعد از هر تغییر: import کل ماژول‌ها + راه‌اندازی dispatcher را چک کن، و در صورت امکان فلوی منطقی را با SQLite تست کن.

## ویژگی‌های v1.9 (آپدیت بزرگ)

- **اتحادها:** بخش دیپلماسی → «اتحاد». فقط کشورهای VIP (`is_vip`) می‌توانند اتحاد بسازند و حداکثر `ALLIANCE_MAX_MEMBERS=6` کشور بیاورند. پنل: «اتحاد من» (جزئیات + مالک + مفاد + خروج) و «ایجاد اتحاد جدید». مدل‌ها: `Alliance`, `AllianceMember`؛ repo: `alliances`؛ handler: `alliance.py`.
- **نامه‌رسان:** بخش دیپلماسی → «ارسال نامه» سه گزینه: نامه به یک کشور / به چند کشور / صندوق پستی. زیر هر نامه دکمه‌ی «پاسخ به نامه»؛ صندوق پستی وضعیت پاسخ‌داده/بی‌پاسخ را نشان می‌دهد. مدل `Letter`؛ repo `letters`؛ handler `mail.py` (مالک `dip:letter`؛ هندلرهای قدیمی نامه از `diplomacy.py` حذف شد).
- **تأسیسات مشترک:** بخش احداث تأسیسات → «تأسیسات مشترک». انتخاب شریک → نوع → درصد شریک → محل → درخواست به شریک؛ با تأیید، هزینه به نسبت تقسیم (سهم شریک = درصد) و بازدهی هر ۲۴ ساعت با همان نسبت بین دو کشور تقسیم می‌شود. ستون‌های `partner_country`,`partner_percent` روی `facilities` (در `_COLUMN_MIGRATIONS`)؛ مدل `JointBuildRequest`؛ سرویس `build_joint_facility`؛ handler `joint.py`.
- **بانک:** بخش اقتصاد → «بانک»: موجودی، بدهی (+پرداخت با تأیید)، انتقال وجه (کشور→مبلغ→تأیید)، وام (غیرفعال). از `budget`/`govt_debt` استفاده می‌کند؛ handler `bank.py`.
- **سرمایه‌گذاری:** بخش اقتصاد → «سرمایه‌گذاری»: داخلی (روی خود) یا خارجی. ۱۰ دسته با درصد سود ۲۴ساعته در `INVESTMENT_CATEGORIES`. اصل سرمایه هنگام ثبت از بودجه کسر می‌شود؛ سود نقدی هر ۲۴ ساعت به **سرمایه‌گذار** می‌رسد (در `scheduler.process_investments`). سرمایه‌گذاری خارجی روی کشور هدف اثر اجتماعی دارد (+رضایت/-بیکاری/-تورم) و خبرش در کانال اقتصاد منتشر می‌شود؛ داخلی منتشر نمی‌شود. مدل `Investment`؛ handler `investment.py`.
- **پیام بازدهی:** هنگام بازدهی ۲۴ساعته‌ی تأسیسات/کارخانه‌ی نظامی/سرمایه‌گذاری، ربات به مالک پیام «سود/تولید + زمان بعدی» می‌فرستد (در `scheduler`).
- **اعلانات (تفکیک):** `/announce متن` → به همه (سازگاری قبلی)؛ `/announce` بدون متن → منوی «به همه / به یک‌چند کشور» (انتخاب چندتایی). `AnnounceForm` در `admin.py`.
- **خبر رفع تحریم:** هنگام لغو تحریم، خبر در کانال دیپلماسی + اطلاع به کشور هدف + لاگ.
- **کول‌داون‌ها (v1.9):** دیدار حضوری ۱/۳ساعت، بیانیه ۱/۱۰دقیقه، تماس ۱/۳۰دقیقه (با repo `cooldowns`). محدودیت ساخت به‌تفکیک نوع در ۱۲ ساعت: معدنی ۵، فولاد ۲، دکل نفت/گاز ۳ (مشترک)، کارخانه نظامی ۲ (`BUILD_LIMIT_GROUPS`, `MIL_FACTORY_BUILD_LIMIT`).
- **انحصار نشست‌ها (v1.10):** کشور تا پایان نشست/تماس فعال نمی‌تواند تماس/دیدار/نشست جدید بسازد یا بپذیرد؛ شامل حالت در پرواز (`get_active_engagement_label`).
- **ممیزی لاگ:** همه‌ی اقدامات کشورها (نامه، بیانیه، قرارداد، بانک، سرمایه‌گذاری، اتحاد، تأسیسات مشترک، اعلان، رفع تحریم و ...) در گروه لاگ ثبت می‌شوند.

## قواعد ظاهری UI (v1.8)

- رنگ دکمه‌ها از طریق فیلد `style` در `InlineKeyboardButton` (Bot API 9.4+) اعمال می‌شود.
  ثابت‌ها در `bot/utils/ui.py`: `STYLE_OK='success'` (سبز)، `STYLE_NO='danger'` (قرمز)، `STYLE_MAIN='primary'` (آبی).
- قرارداد رنگ‌ها: سبز = تأیید/پذیرش/خرید/فروش/احداث/انتخاب‌شده؛ قرمز = لغو/رد/حمله/وضع تحریم/پایان فصل/پایان تماس؛ آبی = دکمه‌های اصلی منو و کنش اصلی هر صفحه؛ بدون رنگ = فهرست‌ها و بازگشت.
- ایموجی نشانگر انتخاب: `PICK_ON='🟢'` / `PICK_OFF='⚪️'` (به‌عنوان fallback برای کلاینت‌های بدون پشتیبانی رنگ).
- سربرگ‌ها با `header(title, emoji)` ساخته می‌شوند (عنوان درشت + خط جداکننده `DIVIDER`).
